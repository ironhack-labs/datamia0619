This is a tracking category
It is used to build and maintain lists of pages—primarily for the sake of the lists themselves and their use in article and category maintenance. They are not part of the encyclopedia\'s categorization scheme. \n
\n\n\n\n\n\n
More information:
\n
\n
  • This category is hidden on its member pages—unless the corresponding user preference is set.
    • \n
  • These categories are used to track, build and organize lists of pages needing "attention en masse" (for example, pages using deprecated syntax), or that may need to be edited at someone\'s earliest convenience.
    • \n
  • These categories also serve to aggregate members of several lists or sub-categories into a larger, more efficient list (discriminated by classifications).
\n
\n
Administrators: Please do not delete this category even if it is empty!
This category may be empty occasionally or even most of the time.
\n

This is a tracking category for CS1 citation templates that have author parameters that contain text that is inappropriate in these parameters. In particular, this category tracks cs1|2 templates that include editor annotation in |authorn=, |lastn=, |firstn= (and aliases). For example, |author2=Green, V. (ed) is inappropriate because editors have their own parameters. Pages in this category should only be added by Module:Citation/CS1.\n

\n
\n

By default, Citation Style 1 and Citation Style 2 error messages are visible to all readers and maintenance category messages are hidden from all readers.\n

To display maintenance messages, include the following text in your common CSS page or your specific skin\'s CSS page (common.css and skin.css respectively):\n

\n
.mw-parser-output .cs1-maint {display: inline;} /* display Citation Style 1 maintenance messages */\n
\n

Even with this css installed, older pages in Wikipedia\'s cache may not have been updated to show these error messages even though the page is listed in one of the tracking categories. A null edit will resolve that issue.\n

To hide error messages:\n

\n
.mw-parser-output .cs1-visible-error {display: none;} /* hide Citation Style 1 error messages */\n
\n

You can personalize the display of these messages (e.g. change the color), but you will need to ask someone who knows CSS or at the technical village pump if you do not understand how.\n
